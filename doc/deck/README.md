# 발표자료 재현 절차

`헬름_IRP세븐서티액티브_발표자료.pptx` 와 `투자제안서_v1.3.md` 는 **같은 한 벌의 숫자**
(`metrics.json`)를 쓴다. **이 브랜치 하나에 엔진·데이터·문서가 모두 들어 있어
워크트리 없이 clone 직후 바로 재현된다.**

> 2026-07 통합: 이전에는 엔진이 `origin/v2-tier2a-freeze` 에만 있고 데이터는
> gitignore 라 워크트리를 만들어야 했다. 지금은 엔진을 병합하고 시세를 박제해
> 이 브랜치가 자체 완결된다.

## 1. 백테스트 재현

```bash
python run_v2.py --end 2026-06-30     # 요약표 + 연도별표를 로그로 출력
```

- 종료일 `2026-06-30` 은 `config.json` 의 `end` 와 같으므로 인자 없이 `python run_v2.py`
  로도 동일하다. **단 `--end` 를 데이터 끝(2026-07-15)까지 주면 다른 수치가 나온다**
  (7월 전반 하락 반영). 문서는 표방 구간 ~2026.06 을 따른다.
- 시세는 박제돼 있으므로(아래 '데이터 스냅샷') 언제 돌려도 같은 결과가 나온다.

## 2. 수치·차트 생성

## 데이터 스냅샷

`datasets/ohlcv/*.parquet` 는 이 저장소에 **박제**돼 있다(2026-07 커밋).

- 데이터 최종 종가일: **2026-07-15**, 문서 산출 종료일: **2026-06-30**
- 박제 이유: 로더가 `auto_adjust=True` 라 배당이 소급 반영돼, 시세를 **언제
  받았느냐에 따라** 2026 후반 배당락 구간 수익률이 달라진다. 그래서 엔진·파라미터가
  같고 2025-12-31 컷까지 일치하는데도 2026 수익률만 갈렸다(팀 내 27.5 vs 29.3 논쟁).
  버그가 아니라 다운로드 시점 차이라, 데이터를 고정하는 것이 유일한 해결이다.
- 데이터를 갱신하려면 **팀 합의로만** 한다. 갱신 시 문서 수치도 함께 바뀐다.

## 알려진 이력

- 제안서 **v1.2 까지의 수치는 시세가 2026-06-15 까지만 적재된 상태**에서 산출된 것이다.
  `python run_v2.py --end 2026-06-15` 로 정확히 재현된다(V1 +44.7% / V2 +27.5%).
  v1.3 부터는 6월 전체가 적재된 데이터 기준이다(V1 +46.9% / V2 +29.3%).
- 2020~2025 연도별 수익과 2025년말 컷 지표는 **두 시점에서 완전히 동일**하다 —
  차이는 오직 6월 후반의 배당락 소급 반영에서 나온다.
- **`CLAUDE.md` 작업 브리핑(2026-07-15)의 동결 수치(CAGR 13.4 / Sharpe 1.18 /
  Calmar 1.05 / 2026 +27.5)도 6/15 스냅샷 값이다.** 데이터 박제 이후 공식 수치는
  6/30 기준(13.6 / 1.20 / 1.07 / +29.3)이며, 브리핑 값과 어긋나면 **이 README 와
  박제 데이터가 기준**이다. 팀 리뷰(2026-07)에서 6/30 유지로 결정했다.
